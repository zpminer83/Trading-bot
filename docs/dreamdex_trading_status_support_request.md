# dreamDEX trading-status support request

This note records read-only questions for the maintainers. It intentionally
contains no account addresses, credentials, request values, or mutation
instructions.

1. Which documented read-only API field is authoritative for market lifecycle
   and trading-enabled status for a market?
2. Is there a documented, stable read-only endpoint or contract view for
   place-operation support and cancel-operation support?
3. Which response schema and identity fields make an authenticated account
   snapshot authoritative for balances, open orders, and fills?
4. What is the documented chain identifier and RPC method contract for the
   public Somnia Mainnet endpoint?

Until these questions have source-confirmed answers, the rehearsal remains
fail-closed and does not form a candidate for signing or submission.
